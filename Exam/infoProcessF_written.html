01.HTML 이란?

HTML (HyperText Markup Language) 은 웹 페이지의 형태를 기술하기 위한 마크업 언어를 의미합니다.
웹의 기초적인 구성 요소로, 웹 페이지의 구조와 콘텐츠의 의미를 정의합니다.

참고하기

1HyperText

웹 페이지를 다른 페이지로 연결하는 링크를 뜻합니다.

2HTML 의 최초 제안자는 유럽 입자 물리 연구소 CERN 의 물리학자 "팀 버너스리 (Tim Berners-Lee)" 입니다.

"팀 버너스리"는 연구원들의 문서 공유 체계를 시작으로 인터넷 기반 하이퍼텍스트를 개발하였고, 이후에 TCP/IP 통신규약을 만든 전산학자 "빈트 서프 (Vinton Gray Cerf)" 와 함께 인터넷의 아버지로 불립니다.

3마크업 언어

웹 브라우저가 해독할 수 있도록 문서를 논리적으로 구조화하는 규칙을 정의한 언어를 의미합니다.

4웹의 기초적 구성요소 3가지

HTML (구조) '무슨 의미인가?'
웹 페이지의 구조와 콘텐츠의 의미를 정의합니다. ☛ 건물의 뼈대 / 사람의 골격
JavaScript (동작) '어떤 동작을 하는가?'
문서와 사용자의 상호작용을 위한 웹 페이지의 기능과 동작을 정의합니다. ☛ 엘레베이터 등 건물의 기능 / 사람의 근육조직 및 신체기관
CSS (표현) '어떻게 보이는가?'
웹 페이지에 관한 다양한 스타일을 정의합니다. ☛ 건물의 디자인 / 사람의 옷

02.HTML 기본 구성

1요소 (element)

HTML 문서를 구성하는 개별적인 명령어를 말하며, 시작 태그부터 종료 태그까지를 포함합니다.
대부분의 요소들은 '태그, 속성, 내용'으로 이루어져 있고, 중첩 구조를 가질 수 있습니다.
내용과 종료 태그 없이 단독으로 사용하는 요소를 '빈 요소 (empty elements)'라고 합니다.

참고하기

1태그 (tag)

요소의 시작과 끝을 표시하는 마크업 기호를 말하며, 각 괄호 < > 를 사용하여 나타냅니다.

2요소의 중첩 구조

< html > 요소는 < head > 와 < body > 요소로 구성되어 있습니다.
< head > 요소 안에는 < meta >, < title >, < link >, < style >, < script > 등의 요소가 포함될 수 있습니다.

중첩구조 보기
 
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>탭에 보여지는 문서의 제목</title>

    <!-- 링크로 css 파일 연결 -->
    <link rel="stylesheet" href="assets/css/fonts.css">

    <!-- 문서 내에서 CSS 지정 -->
    <style>
        * {
            margin: 0;
            padding: 0;
            box-sizing: border-box;
        }
        body {
            background-color: #f5f5f5;
        }
    </style>
    
    <!-- 스크립트로 js 파일 연결 -->
    <script src="../assets/js/common.js"></script>
</head>
<!-- //head -->
    
<body>
    <h1>브라우저 프레임 안에 보이는 제목</h1>
    <p>문단 내용 1</p>
    <p>문단 내용 2</p>
</body>
<!-- //body -->
</html>

3빈 요소의 예

< br > 줄바꿈 요소
텍스트에서 줄의 구분이 필요한 내용을 작성할 때 사용합니다.
< img > 이미지 삽입 요소
문서에 이미지를 넣을 때 사용합니다.

요소의 예
html 요소의 예
요소명 설명
<meta> HTML 문서에 대한 설명, 키워드, 문서의 작성자 등의 메타 데이터 요소를 나타냅니다.
<title> 제목 표시줄 또는 페이지 탭에 보이는 HTML 문서의 제목을 설정합니다.
<link> HTML 문서에 외부의 리소스를 연결합니다.
CSS 파일을 HTML 문서에 연결할 때 제일 많이 사용합니다.
<style> HTML 문서 내에서 스타일(CSS) 정보를 입력하기 위해 사용합니다.
<script> HTML 문서 내부에서 자바스크립트 정보를 입력하거나, 외부에서 자바스크립트 파일을 불러오기 위해 사용합니다.

2속성 (attribute)

요소의 동작을 제어하기 위해 태그 내부에 부가적인 설정값을 선언하는 것을 의미합니다.
요소의 기본 기능을 수정하거나 스스로 기능하지 못하는 특정 요소에 기능을 제공하기 위해 사용합니다.
속성은 여는 태그 안에 속성 명 = " 속성 값 " 의 형식으로 기술합니다.

참고하기

1하나의 태그 안에 '속성 명' 과 '속성 값' 은 동시에 여러 개가 존재할 수 있습니다.

<p class = "desc" style = "width: 50px;     color: blueviolet;     border: 10px;">
속성의 예
1lang 속성

HTML 문서에서 사용하는 언어를 설정합니다.
시각 장애인들이 사용하는 스크린 리더 (음성 낭독 프로그램) 은 lang 속성의 언어 설정에 따라 문서를 읽어주므로,
웹 접근성을 지키기 위해 lang 속성은 HTML 문서에서 반드시 정의해야 하는 속성입니다. 

<html lang = "ko">          // 언어를 한국어로 설정 : 숫자를 " 일, 이, 삼 " 으로 읽음
<html lang = "en">          // 언어를 영어로 설정 : 숫자를 " 원, 투, 쓰리 "로 읽음
참고하기

1웹 접근성

장애인, 노약자 등의 사용자가 신체적 차이, 지식의 정도 등에 의한 불편함 없이 웹 사이트의 모든 정보를 이용할 수 있도록 보장하는 것을 말합니다.

2주 언어 설정 이후, 특정 부분에서 언어를 바꿔주어야 한다면 해당 부분만 다른 언어로 지정할 수 있습니다.

<i lang="de">Ich liebe dich</i> // 한글은 어떻게 나오나 보자
2charset 속성

< meta > 태그 안에서 문자 인코딩 (charset) 방식을 설정합니다.
charset 속성의 값으로 주로 UTF-8 을 사용하는데, UTF-8 는 대부분의 문자를 처리할 수 있습니다. 

<meta charset = " UTF-8 ">
참고하기

1인코딩 (encoding)사용자가 입력한 문자나 기호들을 컴퓨터가 이용할 수 있는 신호로 만드는 것을 말합니다.

2viewport현재 화면 상에 표시되고 있는 영역을 의미합니다.

모바일 해상도에 맞게 HTML 문서의 가로 세로 크기를 조절해주는 속성입니다.
웹 페이지를 만들 때 뷰포트 설정을 해 주어야 반응형 페이지를 제대로 동작시킬 수 있습니다. 

<meta name = " viewport "     content="width=device-width,     initial-scale=1.0">

3DOCTYPE 선언

'document type'의 약자로, 웹 문서가 어떤 형식으로 작성되었는지 브라우저에게 알리기 위해 HTML 문서의 최상단에 선언합니다.
이를 통해 브라우저는 문서의 형식과 버전을 확인하고 요소/태그의 지원 여부, 속성 처리 기준에 따라 정보를 표시합니다.
DOCTYPE 선언은 각 브라우저가 HTML 문서의 내용을 올바르게 표시하도록 돕는 역할을 하며, 문서 간의 호환성을 위해 사용합니다. 

 <!DOCTYPE html>          // !DOCTYPE 을 선언하며 시작
    <html lang="ko">         // lang 속성 설정
    <head>
        <meta charset="UTF-8">       // charset 속성 설정
        <title>Document</title>
    </head>
    <body>
        
    </body>
    </html>
참고하기

1브라우저의 종류

Chrome, Edge, Firefox, Opera, Safari, Whale, Vivaldi, Brave 등

2< !DOCTYPE html > 을 선언하지 않으면,

같은 내용의 HTML 문서일지라도 브라우저 환경에 따라 전혀 다른 결과물을 출력할 수 있습니다.
